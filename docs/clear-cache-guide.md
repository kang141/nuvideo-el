# 清除开发缓存指南

## 问题现象

如果你看到控制台有多个版本的日志同时输出，比如：
```
useVideoExport.ts?t=1772633303288:258 [性能监控] 进度: 0.0%
useVideoExport.ts:332 [性能监控] 进度: 25.0%
```

这说明有旧版本的代码还在缓存中运行。

## 解决方法

### 方法 1: 硬刷新（推荐）

1. **Windows/Linux**: 按 `Ctrl + Shift + R` 或 `Ctrl + F5`
2. **macOS**: 按 `Cmd + Shift + R`

### 方法 2: 清除缓存并硬刷新

1. 打开开发者工具（F12）
2. 右键点击刷新按钮
3. 选择"清空缓存并硬性重新加载"

### 方法 3: 完全重启应用

1. 关闭应用窗口
2. 在终端中停止开发服务器（Ctrl + C）
3. 重新运行 `npm run dev`
4. 重新打开应用

### 方法 4: 清除 Electron 缓存

如果上述方法都不行，清除 Electron 的缓存目录：

**Windows:**
```bash
# 删除缓存目录
rmdir /s /q "%APPDATA%\nuvideo"
```

**macOS/Linux:**
```bash
# 删除缓存目录
rm -rf ~/Library/Application\ Support/nuvideo
```

## 验证是否成功

刷新后，控制台应该只有一个版本的日志：
```
[性能监控] 进度: 25.0%, 已渲染: 114/456, 平均FPS: 56.8
[性能监控] 进度: 49.3%, 已渲染: 225/456, 平均FPS: 56.2
[性能监控] 进度: 75.4%, 已渲染: 344/456, 平均FPS: 57.3
```

不应该再看到 `useVideoExport.ts?t=xxxxx` 这样的时间戳版本。

## 开发建议

在开发过程中，如果修改了核心代码：
1. 始终使用硬刷新（Ctrl + Shift + R）
2. 如果使用了 Service Worker，需要在开发者工具中注销它
3. 定期清理浏览器缓存
