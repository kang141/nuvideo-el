# 视频导出卡顿调试指南

## 🔍 调试日志说明

### 1. 性能监控日志（每 2 秒输出）

```
[性能监控] 进度: 45.2%, 已渲染: 1356/3000, 平均FPS: 58.3, 慢渲染: 12, 慢写入: 5, 队列峰值: 23
```

**关键指标：**
- `进度`: 当前导出进度百分比
- `已渲染/总帧数`: 已完成渲染的帧数
- `平均FPS`: 实际渲染速度（应接近目标 60fps）
- `慢渲染`: 渲染耗时 >50ms 的帧数量
- `慢写入`: 写入耗时 >100ms 的帧数量
- `队列峰值`: 帧队列的最大积压数量

### 2. 帧处理日志

```
[导出调试] 帧 1234 渲染耗时 85.32ms (异常慢)
[导出调试] 帧 1234 数据提取耗时 45.12ms (异常慢)
[导出调试] 帧 1234 总处理时间 132.45ms (渲染: 85.32ms, 提取: 45.12ms)
```

**问题定位：**
- 渲染慢 (>50ms): 可能是复杂特效、高分辨率或 GPU 性能不足
- 数据提取慢 (>30ms): 可能是 Canvas 尺寸过大
- 总处理时间 >100ms: 会导致队列积压

### 3. 队列状态日志

```
[导出调试] 队列积压严重: 45 帧，可能导致卡顿
[导出调试] 队列长度: 23, 当前帧: 1500
```

**问题分析：**
- 队列长度 <10: 正常，写入速度跟得上渲染
- 队列长度 10-30: 轻微积压，可接受
- 队列长度 >30: 严重积压，FFmpeg 编码速度跟不上

### 4. FFmpeg 写入日志

```
[FFmpeg Export] 缓冲区接近满载: 52428/65536 (80.0%)
[FFmpeg Export] 缓冲区已满，等待 drain 事件...
[FFmpeg Export] drain 完成，耗时 234.56ms
[FFmpeg Export] 帧 1234 写入耗时 245.67ms (异常慢)
```

**问题定位：**
- 缓冲区满载: FFmpeg 编码速度慢，无法及时消费数据
- drain 耗时长: 编码器性能瓶颈
- 写入耗时 >50ms: 需要优化编码参数

### 5. FFmpeg 编码器日志

```
[FFmpeg Export] 进度: frame=1234, fps=45.2, speed=0.75x
[FFmpeg Export] 性能警告: buffer underflow detected
```

**关键指标：**
- `fps`: FFmpeg 实际编码速度（应接近目标 60fps）
- `speed`: 编码速度倍率（1.0x = 实时，<1.0x = 慢于实时）
- 性能警告: 编码器遇到性能问题

## 🎯 卡顿原因诊断

### 场景 1: 渲染慢导致卡顿

**症状：**
```
慢渲染: 150+
平均FPS: 35.2
队列峰值: 5
```

**原因：** 渲染本身就慢，不是写入问题

**解决方案：**
1. 降低导出分辨率（选择"流畅"质量）
2. 简化特效和滤镜
3. 检查 GPU 驱动是否最新

### 场景 2: FFmpeg 编码慢导致卡顿

**症状：**
```
慢写入: 200+
队列峰值: 50+
FFmpeg fps=25.3, speed=0.42x
缓冲区频繁满载
```

**原因：** FFmpeg 编码速度跟不上渲染速度

**解决方案：**
1. 降低编码质量（提高 CRF 值）
2. 使用更快的编码预设
3. 检查是否启用了硬件加速（NVENC）
4. 关闭其他占用 GPU 的程序

### 场景 3: 数据传输慢导致卡顿

**症状：**
```
数据提取耗时频繁 >30ms
写入耗时正常
队列长度波动大
```

**原因：** Canvas 数据提取或 IPC 传输慢

**解决方案：**
1. 降低导出分辨率
2. 确保没有其他程序占用 CPU
3. 检查系统内存是否充足

### 场景 4: 间歇性卡顿

**症状：**
```
大部分帧正常，偶尔出现 >500ms 的慢帧
进度条突然停顿 1-2 秒
```

**原因：** 系统资源被其他进程抢占

**解决方案：**
1. 关闭后台程序（浏览器、杀毒软件等）
2. 确保电源模式为"高性能"
3. 检查磁盘空间是否充足

## 🛠️ 优化建议

### 1. 根据日志调整编码参数

如果看到 `speed < 0.8x`，修改 `electron/main.ts`:

```typescript
// NVENC: 降低质量预设
'-preset', 'p3',  // 从 p5 降到 p3

// libx264: 使用更快预设
'-preset', 'ultrafast',  // 从 veryfast 改为 ultrafast
```

### 2. 增加缓冲区大小

如果频繁看到"缓冲区满载"，修改 `electron/main.ts`:

```typescript
'-thread_queue_size', '2048',  // 从 1024 增加到 2048
'-bufsize', '200M',            // 从 100M 增加到 200M
```

### 3. 调整队列大小

如果队列峰值 >50，修改 `src/hooks/editor/useFFmpegExport.ts`:

```typescript
// 增加队列警告阈值
if (frameQueue.length > 50) {  // 从 30 改为 50
  logger.warn(`队列积压严重: ${frameQueue.length} 帧`);
}
```

## 📊 性能基准

### 正常导出性能指标

- 平均 FPS: 55-60
- 慢渲染: <5%
- 慢写入: <2%
- 队列峰值: <15
- FFmpeg speed: >0.9x

### 可接受性能指标

- 平均 FPS: 45-55
- 慢渲染: 5-10%
- 慢写入: 2-5%
- 队列峰值: 15-30
- FFmpeg speed: 0.7-0.9x

### 需要优化的性能指标

- 平均 FPS: <45
- 慢渲染: >10%
- 慢写入: >5%
- 队列峰值: >30
- FFmpeg speed: <0.7x

## 🔧 调试步骤

1. **开始导出**，打开开发者工具（F12）查看控制台

2. **观察性能监控日志**（每 2 秒输出）
   - 记录平均 FPS
   - 记录慢渲染/慢写入次数
   - 记录队列峰值

3. **查找异常日志**
   - 搜索"异常慢"关键词
   - 搜索"缓冲区满载"
   - 搜索"性能警告"

4. **分析 FFmpeg 输出**
   - 查看编码速度（speed）
   - 查看实际 fps
   - 查看是否有错误信息

5. **根据诊断结果优化**
   - 参考上面的场景分析
   - 应用对应的解决方案
   - 重新测试

## 📝 报告问题时请提供

1. 完整的控制台日志（从开始导出到结束）
2. 导出配置（分辨率、质量、时长）
3. 系统信息（CPU、GPU、内存）
4. 性能监控的关键数据
5. 卡顿发生的具体时间点

这些信息将帮助快速定位问题根源。
