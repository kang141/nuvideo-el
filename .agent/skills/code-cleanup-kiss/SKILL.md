# KISS Code Cleanup & Refactor Skill

---

name: code-cleanup-kiss
description: 恪守 KISS (Keep It Simple, Stupid) 原则，消除过度工程化，崇尚简洁与可维护性。

---

## 核心哲学

- **大道至简**：如果一段代码可以用 10 行写清楚，绝不使用 20 行。
- **直击本质**：剖析问题的第一性原理，去除不必要的防御性设计和中间抽象层。
- **事实为本**：代码应直观反映其功能，避免晦涩的命名或过度通用的设计。

## 职责

1. **清理冗余**：识别并删除未使用的代码、导入、变量以及重复的逻辑。
2. **重构提纯**：将复杂的条件嵌套、冗长的函数重构为清晰、扁平的结构。
3. **消除过度设计**：
   - 移除不必要的接口、泛型代理和深层继承。
   - 质疑“为了未来可能的需求”而添加的复杂逻辑。
4. **性能微调**：在保持简洁的前提下，优化核心循环和内存占用。

## 执行准则

- 在进行重构前，必须向用户阐述“为什么现在的代码太复杂”以及“简化后的好处”。
- 始终保持代码的可读性，简洁不代表牺牲可理解性。
